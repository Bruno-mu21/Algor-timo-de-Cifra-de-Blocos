O trabalho consistiu no desenvolvimento de um sistema distribuído de processamento de pontos 3D, no qual o servidor gera dados, coordena a distribuição dos blocos de processamento e reúne os resultados enviados pelos clientes, que por sua vez utilizam CUDA ou OpenCL para realizar as transformações. O processo inicia no servidor, onde o operador informa os valores de rotação, translação e escala nos eixos x, y e z. A partir desses parâmetros, o servidor gera uma matriz contendo 5 milhões de pontos aleatórios (x, y, z), além de uma segunda matriz de mesmo tamanho inicializada com coordenadas (0, 0, 0), destinada a armazenar o resultado final. Também é gerada a matriz 4x4 de transformação, que combina os parâmetros fornecidos. Para controle interno, o servidor cria ainda uma matriz booleana de 500 posições, inicialmente preenchida com falso, usada para marcar quais blocos (cada bloco contendo 10 mil pontos) já foram processados. Definida a estrutura, o servidor inicia o processamento distribuído e posiciona o índice de trabalho em 0.

O protocolo de processamento segue uma interação contínua entre servidor e múltiplos clientes. Primeiro, o servidor envia aos clientes um aviso de início de processamento, junto com a matriz 4x4 de transformação. Ao receber o aviso, cada cliente solicita um pacote de dados. Quando o servidor recebe essa solicitação, verifica se a posição atual já atingiu os 5 milhões de pontos. Caso tenha atingido, responde com código 0, indicando que o processamento terminou; caso contrário, responde com código 1, informando a posição do próximo bloco e enviando 10 mil pontos correspondentes àquele trecho da matriz original. O cliente então aplica a transformação usando CUDA ou OpenCL e retorna ao servidor o bloco processado, indicando também a posição correspondente. O servidor recebe os resultados e armazena os novos pontos na matriz inicialmente zerada, na posição correta, e marca o bloco como processado na matriz booleana. O cliente volta a solicitar o próximo bloco, repetindo o ciclo até que todos os blocos tenham sido processados. Quando todos os valores da matriz booleana forem verdadeiros, o servidor conclui que o processo está completo e encerra a operação.

Além disso, o trabalho prevê experimentação com diferentes tamanhos de blocos de processamento — 10K, 20K, 50K e 100K pontos por pacote — permitindo avaliar desempenho, velocidade de distribuição, paralelismo e eficiência tanto no servidor quanto nos clientes. Dessa forma, o projeto aborda conceitos de computação distribuída, processamento paralelo com GPU, coordenação de tarefas e otimização baseada em blocos de dados.
